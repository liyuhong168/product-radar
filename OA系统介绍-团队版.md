## 一、OA是什么

OA（运营门户）是把三店（322·007·027）日常运营需要用到的**数据工具集中在一个页面上**，不用在多个网址之间来回切换。

**一句话：** OA = 一个导航页 + 三个核心工具。

```
┌─────────────────────────────────────────────────────┐
│  📋 Amazon项目运营OA                                │
│  322 · 007 · 027                                    │
├─────────────┬───────────────────────────────────────┤
│  📡 跨境雷达 │       [iframe 内容区]                 │
│  🎯 选品平台 │       当前显示的板块内容               │
│  📦 补货跟进 │                                       │
└─────────────┴───────────────────────────────────────┘
```

**访问地址：** https://liyuhong168.github.io/product-radar/

## 二、三大板块详解

### 📡 板块1：跨境雷达

| 项目 | 说明 |
|------|------|
| **定位** | 24小时跨境电商情报聚合 |
| **更新频率** | 每天09:00自动更新 |
| **数据量** | ~363条信号/天，聚类10+跨源事件 |
| **数据来源** | AMZ123、亿恩网、雨果跨境等30+中文跨境资讯源 |

**能做什么：**
- 看今日跨境精选Top 10（评分排序，一眼知道重要事件）
- 看今日行动清单（需要你立刻处理的事）
- 看政策日历（哪些法规即将生效、截止日）
- 看跨境信号流（平台动态、物流、选品趋势）

**谁用：** Lee（战略决策 + 合规风控）

**典型场景：** 早上打开OA → 点「跨境雷达」→ 看精选Top 10有没有跟自己品类相关的事件 → 看行动清单有没有要处理的事。

### 🎯 板块2：选品平台

**核心功能：选品全流程从发现到决策，一个页面搞定。**

选品平台内部又分4个子板块（Tab切换）：

| Tab | 功能 | 关键数字（今日） |
|-----|------|----------------|
| 🔍 趋势发现 | AI每天分析趋势 → 推荐4个关键词+选品理由 | 今日4个 |
| 📡 雷达扫描 | 自动扫描26个Amazon UK品类，筛选通过的产品 | 今日46个 |
| 📅 节日选品 | 62个英国节日事件 + 三种物流截止日计算 | 未来62个事件 |
| 📋 选品看板 | 所有候选品的管理状态（待评估→找供应商→已采样→已上架→不考虑） | — |

#### 🔍 趋势发现

```
08:40 cron触发AI分析
  → 搜 TikTok/Google/Reddit 热点
  → 提取有潜力的选品关键词
  → 搜 Amazon UK 是否有对应产品
  → 出选品理由 + 竞争分析 + 1688采购链接
  → 展示在平台首页
```

**每个发现关键词展示：** 趋势评分、竞争评分、利润评分、Amazon和1688链接一键跳转。

**谁用：** Lee为主（决策），刘杨为辅（1688找货源）

#### 📡 雷达扫描

```
09:10 + 14:00 自动运行
  → 扫 Amazon UK 26个品类 New Releases / Bestsellers / Movers
  → 自动过滤：价格£6-15 / 利润≥20% / 重量≤200g / 评论≤200
  → 自动排除：电器/灯具/液体/电池/蓝牙/服装/玩具等
  → 结果展示在平台
```

**每个产品卡片展示：** 主图、标题、价格、利润、评分、BSR、NEW标记。

**谁用：** Lee + 刘杨（快速浏览新品，感兴趣的标记状态）

#### 📅 节日选品

内置英国全年节日日历 + 三种物流模式倒计时：

| 方式 | 总周期 | 当前截止日算法 | Lee偏好 |
|------|--------|---------------|---------|
| ✈️ 空运 | 16+14=30天 | 节日前30天截止 | ✅ 主力 |
| 🚆 卡航/快铁 | 33+14=47天 | 节日前47天截止 | ✅ 主力 |
| 🚢 海运 | 63+14=77天 | 节日前77天截止 | ❌ 不做 |

**每个节日展示：** 名称、日期、剩余天、节前物流截止日、推荐关键词、推荐产品SKU、1688采购链接。

**谁用：** Lee（选品方向决策）

#### 📋 选品看板

三列看板管理候选品状态：

```
📥 收件箱（新品自动流入）→ ⭐ 值得做（手动移入）→ ✅ 已验证（手动移入）
```

**每个卡片：** 来源标记（趋势发现/雷达扫描/节日选品）、标记日期、状态切换。

**谁用：** Lee + 刘杨（协作管理选品进度）

### 📦 板块3：补货跟进

| 项目 | 说明 |
|------|------|
| **定位** | 库存监控 → 补货建议 → 可操作清单 |
| **更新** | 周一/四 08:00自动更新 |
| **覆盖** | 007店 + 027店（所有在售ASIN） |
| **数据源** | 领星ERP PP缓存 + 7天历史导出备选 |

**能做什么：**
- 看哪些SKU需要补货（按FBA可售天数升序排列）
- 看断货风险等级（🔴紧急 / 🟠近期 / 🟡正常）
- 看补货建议数量和建议物流方式
- 看30天销售趋势图

**谁用：** 阿豪 + 淇淇（运营团队，执行补货操作）

**典型场景：** 周一早上 → 打开OA点「补货跟进」→ 看列表页按断货风险排序 → 对紧急SKU执行补货 → 对近期SKU准备下单。

## 六、常见问题

**Q：为什么某天没有新品？**
A：三种可能：(1) 当天Amazon品类页面没变化 (2) 所有品都通过了过滤规则 (3) 扫描超时。看飞书推送日志确认。

**Q：为什么不做电器/灯具？**
A：体积大→FBA费用高，退货率高→利润低，不符合£6-15轻小件策略。

**Q：数据会不会不准？**
A：选品数据每天自动更新。补货数据来自领星ERP（实时的），分析用的是近7天缓存。

**Q：我能改什么？**
A：选品看板中的产品状态可以改（待评估→找供应商→已采样→已上架→不考虑）。其他数据由系统自动管理。

**Q：手机能看吗？**
A：可以。OA支持手机端自适应布局，侧边栏变成汉堡菜单，表格可横向滚动。
