## 一、OA是什么

OA（运营门户）是Amazon英国站三店（322·007·027）的日常运营工具平台，把分散在多个网站的数据工具集中到一个页面。

**一句话：** OA = 一个导航页 + 三个核心板块。

```
┌─────────────────────────────────────────────────────┐
│  📋 Amazon项目运营OA                                │
│  322 · 007 · 027                                    │
├─────────────┬───────────────────────────────────────┤
│  📡 跨境雷达 │       [iframe 内容区]                 │
│  🎯 选品平台 │       当前显示的板块内容               │
│  📦 补货跟进 │                                       │
└─────────────┴───────────────────────────────────────┘
```

**访问地址：** https://liyuhong168.github.io/product-radar/

## 二、三大板块详解

---

### 📡 板块1：跨境雷达

**定位：** 24小时跨境电商情报聚合

**数据来源：** AMZ123、亿恩网、雨果跨境等30+中文跨境资讯源，每日自动采集聚合

**更新频率：** 每天09:00自动更新，约363条信号/天，聚类10+跨源事件

**主要功能：**

| 功能区块 | 说明 |
|---------|------|
| 今日精选 Top 10 | 按评分排序，一眼看清最重要的跨境事件 |
| 今日行动清单 | 列出需要立即处理的事项（如合规截止、政策变更） |
| 政策日历 | 展示即将生效的英国/EU法规及其截止日期 |
| 跨境信号流 | 按主题分类（平台动态、物流、政策、选品趋势等） |

**适用场景：** 每日早间快速了解跨境行业动态，掌握合规风险和政策变化，辅助战略决策。

---

### 🎯 板块2：选品平台

**定位：** 选品全流程工具，从趋势发现到产品筛选到状态管理，一个页面完成。

选品平台内部包含4个Tab：

#### 🔍 趋势发现

数据流：

```
08:40 AI趋势分析
  → 搜索 TikTok/Google/Reddit/热点话题
  → 提取有潜力的选品关键词
  → 搜索 Amazon UK 是否有对应产品
  → 输出选品理由 + 竞争分析 + 利润测算
  → 展示在平台首页
```

**每个发现关键词展示：** 趋势评分、竞争评分、利润评分、Amazon 搜索链接、1688采购链接。

**核心价值：** 用AI从海量趋势中筛选出可落地的选品方向，直接链接到采购和销售渠道。

---

#### 📡 雷达扫描

数据流：

```
09:10 + 14:00 自动运行
  → 扫描 Amazon UK 26个品类（New Releases / Bestsellers / Movers）
  → 自动过滤：价格 £6-15 / 利润≥20% / 重量≤200g / 评论≤200
  → 自动排除：电器/灯具/液体/电池/充电/蓝牙/服装/玩具/化妆品等
  → 结果展示在平台
```

**每个产品卡片展示：** 主图、标题、价格、利润、评分、BSR、NEW标记（当日首次出现）。

**核心价值：** 零人力成本自动扫描Amazon UK新品，将符合轻小件策略的产品筛选出来，避免人工逐页浏览。

---

#### 📅 节日选品

内置英国全年节日日历，结合三种物流模式计算备货截止日：

| 物流方式 | 总周期 | 选品截止倒计时 | 说明 |
|---------|--------|--------------|------|
| ✈️ 空运 | 16天运输+14天缓冲 = 30天 | 节日前30天截止 | 常规选品主力 |
| 🚆 卡航/快铁 | 33天运输+14天缓冲 = 47天 | 节日前47天截止 | 常规选品主力 |
| 🚢 海运 | 63天运输+14天缓冲 = 77天 | 节日前77天截止 | 不适合轻小件模式 |

**每个节日展示：** 名称、日期、剩余天数、各物流截止日、推荐关键词、推荐产品SKU、1688采购链接。

**核心价值：** 提前规划节日选品方向，避免错过备货窗口。

---

#### 📋 选品看板

三列看板管理所有候选品状态：

```
📥 收件箱 → ⭐ 值得做 → ✅ 已验证
（新品自动流入）  （手动移入）  （手动移入）
```

**每个卡片展示：** 来源标记（趋势发现/雷达扫描/节日选品）、标记日期、状态切换按钮。

**附加功能：** 搜索筛选、导出CSV、状态持久化。

**核心价值：** 追踪每个选品从发现到决策的完整进度，避免漏掉有价值的候选品。

---

### 📦 板块3：补货跟进

**定位：** 库存监控 → 补货建议 → 可操作清单

**数据来源：** 领星ERP产品表现数据（PP缓存）+ 近7天历史导出（备选）

**更新频率：** 周一/四 08:00自动更新，覆盖007店+027店所有在售ASIN

**主要功能：**

| 功能 | 说明 |
|------|------|
| 库存预警列表 | 按FBA可售天数升序排列，断货风险一目了然 |
| 断货风险等级 | 🔴 紧急 / 🟠 近期 / 🟡 正常，颜色标记快速识别 |
| 补货建议 | 建议补货数量 + 建议物流方式 |
| 销售趋势 | 30天趋势图，辅助判断补货量 |

**核心价值：** 降低断货风险，减少紧急空运成本，提高库存周转效率。

## 四、技术架构（简版）

### 3+1混合架构

| 板块 | 所属仓库 | 访问方式 | 数据源 |
|------|---------|---------|--------|
| 门户 | product-radar | 根目录 index.html | 生成器内嵌 |
| 选品平台 | product-radar | platform.html | 本地 data/ + 每日扫描 |
| 补货跟进 | product-radar | output/analysis/ | 领星ERP（本地） |
| 跨境雷达 | kj-news-radar（独立仓库） | iframe 直链 | 新闻/政策/RSS |

### 设计系统

所有板块共享同一份 `oa-theme.css`（~750行），统一颜色、排版、间距、组件样式。改一个CSS变量，全站同步更新。

### 部署方式

- **源文件**（.py/.sh/.json/.md）：通过 git push 部署
- **产物文件**（HTML/JS）：通过 GitHub API 直接推送

## 五、数据安全

OA系统部署在 GitHub Pages（公开站点），包含内部运营数据的页面已做脱敏处理：

| 数据类别 | 处理方式 |
|---------|---------|
| 售价、评分、BSR、评论数 | ✅ 公开保留（Amazon Listing 页公开可见） |
| 趋势分析、需求日历、补货建议 | ✅ 保留（功能价值高，不含内部数据） |
| 毛利率、月销量、库存数量 | 🚫 已脱敏移除 |
