# 🛰️ 选品雷达平台 — 功能介绍

> 自动扫描 Amazon UK 新品/热销/飙升趋势，结合多源信号（TikTok、Google、Reddit等），每天两次自动产出候选产品，部署到网页端供团队查看。

**访问地址：** https://liyuhong168.github.io/product-radar/platform.html

---

## 一、整体架构

```
定时触发（每天09:10 + 14:00）
    ↓
run_scan_v2.py (多源扫描引擎)
    ↓
数据存入 data/channels/ + data/discovery/
    ↓
generate_platform.py (生成静态HTML)
    ↓
自动部署到 GitHub Pages (公开可访问)
```

---

## 二、五大核心板块

### 🔍 1. 趋势发现（Discovery）

**做什么：** 每天早上的 AI 趋势分析 → 找出有潜力的选品关键词，自动搜索 Amazon UK 是否有合适的产品。

**数据来源：**
- AI 自动分析（每天08:40 cron触发，搜索TikTok/Google/热点话题）
- AnySearch 多渠道趋势信号
- 季节性关键词预测

**你能看到的：**
- 每个发现关键词的**趋势分数**和**竞争评分**
- 趋势方向（上升/稳定/下降）
- 该关键词下的 Amazon UK 候选产品
- **Amazon 搜索链接** 和 **1688 采购链接**（点开直接看）

**核心价值：** 在别人还没发现之前，提前切入有潜力的品类。

---

### 📡 2. 雷达扫描（Radar）

**做什么：** 固定品类扫描 — 每天自动扫描选定的 Amazon UK 品类页面（New Releases / Bestsellers / Movers & Shakers），自动过滤符合条件的产品。

**扫描品类（26个）：**
宠物用品、厨房、DIY工具、花园、汽车配件、运动户外、办公用品、浴室配件、收纳整理、清洁工具、手工 craft、文具、家居、床上用品、旅行配件、派对用品、手机配件、科技配件 等

**自动过滤规则：**
| 规则 | 说明 |
|------|------|
| 价格区间 | £6–15 |
| 最小利润率 | ≥ 20% |
| 最大重量 | ≤ 300g（轻小件） |
| 评论数 | ≤ 200（蓝海信号） |
| 评分 | ≥ 3.5 |
| ⛔ 硬禁止 | 电器/灯具/液体/电池/充电/蓝牙/服装/玩具/化妆品/食品 等 |

**你能看到的：**
- 产品卡片（主图、标题、价格、评分、评论数、BSR排名）
- 预估利润率和利润额
- 新品/重复标记（今日第一次扫到的标 `🆕 NEW`）
- 按状态筛选（待评估 / 找供应商 / 已采样 / 已上架 / 不考虑）
- 按月/日切换历史数据

**核心价值：** 自动化替代人工枯燥的翻页扫描，每天筛出蓝海候选品。

---

### 📅 3. 节日选品（Festival Planner）

**做什么：** 基于英国节日日历，提前计算不同物流方式的选品截止日期，推荐节日相关选品关键词。

**三种物流模式：**
| 方式 | 总周期 | 适合 |
|------|--------|------|
| ✈️ 空运 | 16天 | 中高价/紧急补货 |
| 🚆 卡航/快铁 | 33天 | 主力方式（Lee常用） |
| 🚢 海运 | 63天 | 大件/低价，Lee不做 |

**你能看到的：**
- 未来90天内的英国节日/事件列表
- 每个事件推荐的选品关键词
- 关键词的 Amazon 搜索链接
- 距离当前还剩多少天

**核心价值：** 提前47天（卡航）锁定节日选品窗口期，不错过季节性热销。

---

### 📋 4. 选品看板（Kanban）

**做什么：** 统一管理所有选品候选 → 跟进状态追踪。

**三列看板：**
| 列 | 含义 |
|----|------|
| 📥 收件箱 | 新扫描出来的候选品，待评估 |
| ⭐ 值得做 | 初步判断值得进一步研究的 |
| ✅ 已验证 | 已通过市场验证的选品 |

**你能操作的：**
- 每个产品上点击状态按钮切换（待评估 → 找供应商 → 已采样 → 已上架 → 不考虑）
- 右上角搜索框筛选
- 导出 CSV 数据
- 点击产品卡片，直接跳转到 **Amazon UK 商品页** 和 **1688 搜同款**

**核心价值：** 选品全流程管理，从"看到"到"上架"每一步都清晰可追溯。

---

### 📊 5. 补充分析工具（后台/命令行）

以下工具不直接显示在网页上，但在后台自动运行或可手动调用：

| 工具 | 功能 |
|------|------|
| **Signal Fusion** (`signal_fusion.py`) | 三层信号融合：趋势分 + 供需缺口 + 利润窗口，综合打分0–100 |
| **Gap Opportunity** (`gap_opportunity.py`) | 识别"高热度 + 低供给"的品类缺口，自动生成选品建议 |
| **Market Intelligence** (`market_intelligence.py`) | 供需比分析 + 趋势背离检测（需求涨但供给还没跟上的品类） |
| **Deep Competitor** (`deep_competitor.py`) | 输入关键词 → 分析竞品格局（CR5集中度、新品占比、价格带分析） |
| **Review Analyzer** (`review_analyzer.py`) | 分析差评主题 → 发现差异化切入点 |
| **Competitor Monitor** (`competitor_monitor.py`) | 监控已选品类的竞品变化（价格/排名波动告警） |
| **Selection Report** (`selection_report.py`) | 每日选品简报生成 |
| **Profit Calculator** (`calc_profit.py`) | 统一利润计算（含 VAT/佣金/FBA/广告/退货） |
| **Season Engine** (`season_engine.py`) | 英国季节性事件预测 + 备货截止日计算 |
| **Success Tracker** (`success_tracker.py`) | 统计选品转化率：发现数 → 跟进数 → 上架数 |

## 三、自动推送（飞书群）

系统每天自动推送到 **工作助理群**：

| 推送 | 时间 | 内容 |
|------|------|------|
| 🔍 选品发现 | 08:40 | AI趋势分析 → 新关键词 + 候选品 |
| 📡 雷达上午扫描 | 09:10 | 品类自动扫描结果（扫了多少、通过多少、新品多少） |
| 📡 雷达下午扫描 | 14:00 | 同上（下午再扫一轮） |
| 📋 选品日报 | 18:00 | 当日选品汇总 |
| 🎯 季节性备货提醒 | 周一/四 09:20 | 节日选品窗口提醒 |

---

## 四、如何使用

1. **每天看推送** — 工作助理群会收到扫描结果和发现
2. **点开网页** — https://liyuhong168.github.io/product-radar/platform.html 查看完整数据
3. **趋势发现 → 看关键词热度 → 点 Amazon 链接看竞品**
4. **雷达扫描 → 看产品卡片 → 标记状态（想不想做）**
5. **节日选品 → 看最近的节日 → 提前选品备货**
6. **看板 → 统一追踪每个选品的进度状态**

---

> 📌 **注意：** 选品平台是辅助发现工具，最终选品决策仍需结合供应链成本、市场验证和团队判断。
> 平台不做电器/灯具/液体类产品，专注轻小件 £6–15 蓝海路线。
